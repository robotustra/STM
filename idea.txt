 Новые идеи, по поводу стэйт машин.

 Допустим, имеем элементарный цикл, микросостояний, например:

 Init -> State1 -> State2 -> ... -> StateN -^

 Так как у нас нет никакого ветвления между состояниями, то количество бит, используемое для описания одного макросостояния равно 0.

 Допустим, у нас есть свитч с 2 возможными выходами, тогда, количество бит, описываемое
 2 макросостояния системы равно 1.

 В общем случае, количество бит, нужное для конфигурации системы равно 

 	NB = log_2(NS)

 NB - количество бит, необходимое для различения сотояний.
 NS - число макро состояний.

 NB 	NS
 0		1
 1		2
 2		3
 2 		4
 3		5
 3		6
 3		7
 3		8
 4		...

 Но для удобства программирования удобне наверное для каждого бинарного расщепления использовать один бит. И более того, использовать ТОЛЬКО бинарные расщепления.


 Например

 Init -> State1 -> State2 -> State3 -> .... -> StateN -^
      \-> State2.1 -> State2.2 -^

 Возможен возврат в первоначальное состояние, или слияние состояний.
